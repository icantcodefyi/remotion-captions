# MeowCap Blog Style Guide

## Voice

- Practical and direct.
- Written for creators, editors, agency operators, and in-house video teams.
- Declarative over promotional.
- No exclamation marks.
- Avoid stacking multiple em-dash constructions in a single sentence.

## Required Signals

- Name the reader by role in the opening paragraph.
- Include at least one concrete example that references the MeowCap workflow.
- End with a specific next step that points to the studio or a sibling article.
- Prefer operational detail over generic encouragement.

## Structure

- One H1 only: the title from frontmatter.
- Four to seven H2 sections.
- Use H3 only when a list is not enough.
- Paragraphs should stay between two and four sentences.
- Default target length: 1,100 to 2,200 words.

## Banned Phrases

- delve
- tapestry
- in today's fast-paced
- navigating the landscape
- it's important to note
- unleash
- unlock
- game-changer
- seamless
- revolutionize

## MeowCap Example Rule

Every draft must include at least one example built around the actual MeowCap flow:

1. Upload a clip.
2. Transcribe or align an approved script.
3. Adjust style or position.
4. Export SRT or JSON for the downstream edit.
